# NetOps Studio — Mock

Тестовая среда для красивой визуализации топологии и детальной информации устройств (Cisco / Huawei / Eltex) **без реального подключения**.
Все данные лежат и редактируются в `devices.txt` (один JSON-объект).

## Быстрый старт

```bash
python3 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python app.py
```
Откройте http://localhost:5000

## Структура
- `app.py` — Flask API и страницы (логин, дашборд, карточка устройства)
- `templates/` — HTML-шаблоны (Jinja2)
- `static/` — CSS/JS. Включает примитивный отрисовщик графа без внешних библиотек.
- `devices.txt` — все устройства, связи и их данные.

## Что можно редактировать
- Метаданные устройства (локация, заметки) — на вкладке "Обзор".
- Интерфейсы (имя, IP, VLAN, скорость, описание, admin_status) — вкладка "Интерфейсы".
- VLAN (ID, имя, тип, список портов) — вкладка "VLAN".

Логи фильтруются по уровню/диапазону дат, маршруты/ARP выводятся в виде таблиц.

## Формат devices.txt (пример коротко)
```json
{
  "devices":[
    {"_type":"device","id":"R1","type":"router","vendor":"Cisco","hostname":"core-r1","management_ip":"10.0.0.1", "...": "..."}
  ],
  "links":[
    {"_type":"link","source":"R1","target":"SW1"}
  ]
}
```

> Поддерживается также JSON Lines формат (по строке на объект) как fallback.
